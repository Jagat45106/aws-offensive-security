Detection context (MITRE ATT&CK T1530 — Data from Cloud Storage & T1083 — File and Directory Discovery): IDOR vulnerabilities in Lambda-backed APIs are extremely common because Lambda functions receive the raw event object and developers often forget to validate that the resource in the path matches the authenticated user. Detection: API Gateway access logs with anomaly detection on userId patterns (sequential scanning). Parameter tampering on export endpoints: alert on export requests targeting tables beyond the expected list. All S3 GetObject calls on export buckets should be logged and reviewed. Remediation: implement resource-based authorization in the Lambda handler, use an allowlist for the table parameter, and require user context to scope exports.
Cleanup: Delete all downloaded files: rm -f users-full.csv payment_methods.csv. The pre-signed export URLs in S3 will expire automatically (typically 15 minutes to 1 hour). Notify the lab instructor that the export endpoint was exercised so they can verify no persistent data was left in the S3 export bucket. All API calls were authenticated and logged in CloudWatch — no API Gateway resources were modified.
